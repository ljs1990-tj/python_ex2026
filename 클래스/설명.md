# 🏛️ 08. 객체 지향 프로그래밍: 클래스 (Class)

자바와 파이썬 모두 객체 지향 언어이지만, 파이썬은 훨씬 더 유연하고 명시적인 방식을 취합니다. 자바의 `this`에 대응하는 `self`의 개념을 이해하는 것이 핵심입니다.

---

## 1. 클래스 구조 비교 (Java vs Python)

자바와 달리 파이썬은 멤버 변수를 미리 선언할 필요가 없으며, 생성자 함수명이 `__init__`으로 고정되어 있습니다.

**[Java 코드]**
```java
public class Robot {
    String name; // 멤버 변수 선언 필요

    public Robot(String name) {
        this.name = name;
    }
}
```

**[Python 코드]**
```python
class Robot:
    def __init__(self, name):
        # 생성자에서 바로 self.name으로 멤버 변수 생성
        self.name = name
```



---

## 2. 'self'의 정체와 역할

자바의 `this`는 암시적으로 존재하지만, 파이썬의 `self`는 **메서드의 첫 번째 매개변수로 반드시 명시**해야 합니다.

- **self**: 인스턴스(객체) 자기 자신을 가리키는 포인터입니다.
- 메서드 호출 시 파이썬이 자동으로 객체 자신을 첫 번째 인자로 넘겨줍니다.

---

## 3. 상속 (Inheritance)

파이썬은 자바와 달리 **다중 상속**을 지원하며, 상속을 명시할 때 클래스 이름 뒤의 괄호`()`를 사용합니다.

**[Python 실습 코드]**
```python
class Unit:
    def __init__(self, name):
        self.name = name

# Unit 클래스를 상속받는 Marine 클래스
class Marine(Unit):
    def __init__(self, name, hp):
        super().__init__(name) # 부모 생성자 호출
        self.hp = hp
```

---

## 4. 접근 제어자 (Private & Public)

자바의 `private`, `protected` 키워드는 파이썬에 존재하지 않습니다. 대신 이름 앞에 **언더바(`_` 또는 `__`)**를 붙이는 관례(Convention)를 사용합니다.

- `_name`: "외부에서 직접 건드리지 마세요"라는 권장(Protected).
- `__name`: "이름을 숨겨서 외부 접근을 막겠다"는 의도(Private / Name Mangling).

---

## 5. 요약 비교표

| 비교 항목 | Java | Python |
| :--- | :--- | :--- |
| **생성자 이름** | 클래스명과 동일 | `__init__` |
| **인스턴스 참조** | `this` (암시적) | `self` (명시적 필수) |
| **다중 상속** | 불가 (인터페이스로 대체) | **가능** |
| **멤버 변수 선언** | 필드 영역에 선언 필수 | 생성자 내에서 즉시 생성 가능 |
| **접근 제어자** | `public`, `private` 등 키워드 | 언더바(`_`) 접두사 관례 |
