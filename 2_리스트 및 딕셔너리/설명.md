# 🧺 05. 파이썬 데이터 꾸러미: 리스트와 딕셔너리

변수 하나에 데이터 하나만 담는 것은 한계가 있습니다. 파이썬은 여러 데이터를 한 번에 담을 수 있는 다양한 **'바구니'**를 제공합니다.

---

## 1. 리스트 (List): 순서가 있는 기차

가장 많이 쓰이는 자료형입니다. 여러 데이터를 일렬로 세워 번호(인덱스)를 붙여 관리합니다.

- **특징**: 데이터의 순서가 있고, 언제든지 내용을 추가하거나 바꿀 수 있습니다.
- **모양**: 대괄호 `[ ]`를 사용합니다.



(리스트 활용 예시)
fruits = ["사과", "바나나", "포도"]
print(fruits[0])      # 결과: 사과 (0번부터 시작!)
fruits.append("수박") # 리스트 끝에 새 데이터 추가
fruits[1] = "딸기"    # 1번 데이터(바나나)를 딸기로 교체

---

## 2. 튜플 (Tuple): 수정할 수 없는 리스트

리스트와 거의 같지만, 한 번 만들면 **내용을 절대 바꿀 수 없습니다.**

- **특징**: 실수로 데이터를 지우거나 바꾸는 것을 방지할 때 사용합니다. (리스트보다 가볍고 빠름)
- **모양**: 소괄호 `( )`를 사용합니다.

(튜플 활용 예시)
coordinates = (10, 20)
# coordinates[0] = 30  -> 에러 발생! (수정 불가)

---

## 3. 딕셔너리 (Dictionary): 이름표가 붙은 상자

사전(Dictionary)에서 단어를 찾으면 뜻이 나오듯, **키(Key)**를 부르면 **값(Value)**이 나오는 구조입니다.

- **특징**: 순서 대신 '이름(Key)'으로 데이터를 찾습니다. 데이터가 방대할 때 검색이 매우 빠릅니다.
- **모양**: 중괄호 `{ }`를 사용하며 `키:값` 형태로 저장합니다.



(딕셔너리 활용 예시)
user = {"name": "김철수", "age": 25, "city": "서울"}
print(user["name"])    # 결과: 김철수
user["age"] = 26       # 값 수정 가능
user["job"] = "개발자"  # 새로운 키와 값 추가

---

## 4. 세트 (Set): 중복을 허용하지 않는 집합

수학의 '집합' 개념과 같습니다.

- **특징**: 데이터의 순서가 없으며, **중복된 값은 자동으로 하나만 남기고 삭제**합니다.
- **모양**: 중괄호 `{ }`를 사용합니다. (키 없이 값만 나열)

(세트 활용 예시)
my_set = {1, 2, 2, 3, 3, 3}
print(my_set) # 결과: {1, 2, 3} (중복 제거)

---

## 5. 💡 언제 어떤 바구니를 쓸까요?

1. **리스트**: 데이터의 순서가 중요하고 자주 내용을 바꿀 때 (출석부, 쇼핑 카트 등)
2. **튜플**: 프로그램이 끝날 때까지 값이 변하지 않아야 할 때 (좌표, 설정값 등)
3. **딕셔너리**: 특정 키워드로 정보를 빠르게 찾아야 할 때 (회원 정보, 단어장 등)
4. **세트**: 데이터의 중복을 제거하거나 집합 연산(교집합, 합집합)이 필요할 때
