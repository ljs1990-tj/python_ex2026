# ⚙️ Java Method vs Python Function: 함수 관점의 정밀 비교

자바 개발자가 파이썬 함수를 다룰 때 구조적으로 혼동하기 쉬운 5가지 핵심 차이점입니다.

---

## 1. 함수의 소속과 호출 방식 (Scope)

- **Java**: 모든 로직은 클래스에 종속됩니다. 호출을 위해 `new` 혹은 `static` 키워드가 필수입니다.
- **Python**: 클래스 외부에서도 존재할 수 있는 **'전역 함수'** 개념이 기본입니다.



---

## 2. 반환값의 개수와 언패킹 (Return Values)

자바는 리턴 타입이 하나로 고정되지만, 파이썬은 쉼표(,)를 통해 사실상 여러 값을 한 번에 보낼 수 있습니다.

**[Python 실습 코드]**
```python
def get_user_info():
    name = "Alice"
    age = 30
    return name, age  # 내부적으로는 (name, age) 튜플 반환

# 호출 시 여러 변수로 즉시 분리 가능 (Unpacking)
user_name, user_age = get_user_info()
```

---

## 3. 다형성 구현: 오버로딩 vs 기본값 인자

파이썬은 메서드 이름이 같고 매개변수가 다른 '오버로딩'을 지원하지 않습니다. 대신 **기본값 매개변수**로 이를 해결합니다.

**[Java: Overloading]**
```java
void printInfo() { ... }
void printInfo(String name) { ... }
```

**[Python: Default Argument]**
```python
def print_info(name="Anonymous"):
    # 매개변수가 없으면 "Anonymous"를 사용함
    print("Name:", name)

print_info()        # 실행 가능
print_info("Bob")   # 실행 가능
```

---

## 4. 함수 자체의 객체화 (First-class Object)

파이썬에서 함수는 변수에 담길 수 있는 '값'입니다. 자바의 익명 클래스나 람다보다 더 직접적으로 함수를 전달합니다.

**[Python 실습 코드]**
```python
def call_twice(func, value):
    # 다른 함수(func)를 인자로 받아 실행 가능
    return func(func(value))

def square(x):
    return x * x

# 함수 자체를 인자로 전달
result = call_twice(square, 3)  # (3*3) * (3*3) = 81
```

---

## 5. 문법적 차이 요약표

| 구분 | Java (Method) | Python (Function) |
| :--- | :--- | :--- |
| **선언 키워드** | 리턴 타입 명시 (`int`, `void` 등) | `def` 키워드 고정 |
| **타입 결정** | 정적 (컴파일 타임) | 동적 (런타임) |
| **다중 반환** | 불가 (객체/배열 활용) | **가능** (튜플 언패킹) |
| **람다 형태** | 인터페이스 구현 필요 | `lambda` 키워드로 즉시 생성 |
